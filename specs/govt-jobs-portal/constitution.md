# Constitution

## Project Purpose

**GovJobs Portal** — a one-stop web application aggregating government job listings from all levels of government (federal, state, council) and other government organisations across Australia.

### Problem
Job seekers must visit 100+ different government job boards to find positions. Existing aggregators (e.g. Seek's government section) are incomplete, stale, and effectively abandoned.

### Solution
A purpose-built aggregator that:
- Automatically discovers and indexes government jobs via official APIs and web scraping
- Provides best-in-class search (ElasticSearch + vector DB for semantic search)
- Sends fine-grained job alert notifications
- Provides preparation content and insider hiring info per department
- Tracks job expiry and re-scans ads before/after expiry dates

### Target Users
- Primary: Job seekers looking for government positions at any level
- Future: Government departments self-publishing roles (paid tier)

## Functional Requirements (Brief)

### Job Aggregation
- Ingest jobs via government-provided REST APIs where available
- Scrape government job boards (federal, state, territory, council, statutory bodies)
- Scrape LinkedIn jobs filtered to government employers (API or scraping)
- Configurable per-source scrape schedules (high-frequency for active boards)
- Track job expiry dates; auto-reschedule scans near/after expiry to detect extensions
- Deduplicate jobs across sources; canonical job record with source attribution

### Search & Discovery
- ElasticSearch for full-text search with filters (level, location, classification, salary, agency)
- Vector DB for semantic/similarity search and offline batch processing (embeddings)
- Faceted filtering, saved searches, search history

### Notifications & Alerts
- Fine-grained job alert subscriptions (keyword, agency, classification, location, salary band)
- Expiry reminders for saved/applied jobs
- Email and/or push notification channels

### Preparation Content
- Blog posts and guides tailored to different government departments and job types
- Insider hiring process information (manually curated, per department)
- Interview preparation, selection criteria writing guides

### Reviews & Ratings
- Scrape Glassdoor reviews for government agencies (where permitted)
- Fallback: build internal review/rating system over time (user-submitted)

### Revenue
- Phase 1: Ad-driven (Google AdSense or similar)
- Phase 2: Paid tier — government departments pay to self-publish and feature listings

## Standards

### Quality
- All scraper modules must have unit tests with fixture HTML/API responses
- ElasticSearch mappings must be versioned; migrations must be reversible
- Scraper schedules must be configurable without code changes (DB-driven config)
- No PII stored beyond what is needed for notifications; GDPR/Privacy Act compliance
- All external HTTP calls (scraping, APIs) must respect robots.txt and rate limits
- Job deduplication logic must have >99% accuracy (tested against known duplicates)

### Architecture Constraints
- Scraping infrastructure must be isolated from the web API (separate ECS services)
- Job ingestion pipeline is async and queue-driven; API never blocks on scraping
- ElasticSearch is the read model; MongoDB is the source of truth
- Vector embeddings are computed in a separate batch worker (not on ingest hot path)

### Review Criteria
- L1 Architecture review: must validate scraper isolation, queue design, search architecture
- L2 Component review: must validate data models, API contracts, scraper plugin interface
- Implementation review: must include integration tests for scraper → DB → search index pipeline
