# Constitution

## Project Purpose

Rentify is a multi-tenanted online rental marketplace platform targeting the Australian market. Tenants (merchants) list physical goods for rent — cameras, baby gear, party hire, tools, construction equipment, sporting goods, and any other leasable asset. Renters (consumers) discover, book, and pay through a unified marketplace with built-in trust (inspections, damage waivers, reviews, deposits).

The platform handles rental-domain complexity that generic e-commerce cannot: time-based availability, inventory lifecycle tracking, condition inspections, security deposit holds/releases, buffer periods between rentals, and dual-sided split payments with commission.

**Key differentiators (from market research):**
- **Compliance-aware rentals**: ACCC consumer guarantees, ATO sharing economy reporting, product safety category gating — embedded as product features, not afterthoughts
- **Category-adaptive trust stacks**: Different verticals (baby, equipment, fashion, party) get different trust requirements configured per category
- **Dual-sided fee model**: Tenant commission + renter booking fee (industry norm per Camplify, Designerex, Sharehire benchmarks)
- **Damage waivers, not insurance**: Platform offers liability reduction arrangements, not regulated insurance products

**Key architectural decisions:**
- Multi-tenant via shared database with `tenant_id` on every row + PostgreSQL RLS
- Availability modelled as blocks (not per-day slots) with exclusion constraints for double-booking prevention
- Payments via Stripe Connect (Express) — deposits use manual-capture pre-auth
- Hybrid attribute model: fixed columns for universal fields, JSONB for category-specific attributes
- Modular monolith to start; extract services when scaling demands it

## Standards

- All monetary values stored as integer cents — no floating-point currency
- All timestamps stored in UTC — timezone on tenant/product for display conversion
- Booking prices locked at creation time — immune to subsequent price changes
- Booking creation is atomic: availability reserved + payment authorized + booking created, or full rollback
- Soft deletes only — no hard row deletion for tenants, users, products, bookings
- Audit log on all financial entity writes
- PCI compliance via Stripe Elements — platform never touches raw card data
- GST (10%) handled correctly: per tenant's GST registration status
- ABN collection mandatory for tenants (ATO reporting regime)
- Trust claims (verification badges, safety-checked labels) must be substantiated by actual processes (ACCC requirement)
- Damage waivers must not be framed as insurance products (ASIC/APRA boundary)
