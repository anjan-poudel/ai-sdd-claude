## Description
Write the unit test suite for the ELA Analyser in `backend/tests/test_ela.py`. Commit test fixture images: a known-authentic JPEG (`authentic.jpg`) and a known-manipulated JPEG with a spliced region (`manipulated.jpg`) under `backend/tests/fixtures/`. Tests must cover the happy path, the in-memory enforcement check, the quality-parameter sensitivity, and the edge-case single-pixel input.

## Acceptance criteria

```gherkin
Feature: ELA tests and fixtures

  Scenario: test_ela.py passes all tests in CI with real fixture images
    Given authentic.jpg and manipulated.jpg are present in backend/tests/fixtures/
    When pytest backend/tests/test_ela.py is run in CI
    Then all tests pass with exit code 0

  Scenario: Manipulated fixture produces a higher sub-score than the authentic fixture
    Given authentic.jpg and manipulated.jpg fixtures
    When run_ela is called on each with default settings
    Then sub_score(manipulated.jpg) > sub_score(authentic.jpg)

  Scenario: ELA tests do not leave temporary files on disk after test execution
    Given the test suite runs in a clean temporary directory
    When pytest backend/tests/test_ela.py completes
    Then no .jpg, .png, or .tmp files exist in /tmp that were not present before the test run
```

## Implementation notes
- Fixtures should be small (< 100 KB) to keep CI fast; generate synthetically if needed using PIL.
- The `manipulated.jpg` fixture can be generated by copying a rectangular patch from one region of an authentic image to another using PIL — this simulates copy-move forgery that ELA can detect.
- Use `pytest.approx` for floating-point score comparisons.
- The test for in-memory enforcement should mock `builtins.open` and assert it is not called with a write mode flag.
