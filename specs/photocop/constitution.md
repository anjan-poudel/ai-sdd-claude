# Constitution

## Project Purpose

**PhotoCop** is a web-based image manipulation and fraud detection platform. It enables any user to upload an image and instantly receive a manipulation probability score, a heatmap highlighting suspicious regions, and a structured JSON report containing EXIF metadata and analysis results.

**Problem solved**: Users cannot easily verify whether a photo has been edited, forged, or AI-generated. PhotoCop provides forensic-grade image analysis in a simple browser interface — no specialist knowledge required.

**Target users**: General public, journalists, insurance professionals, researchers — anyone who needs to verify image authenticity.

## Tech Stack

- **Frontend**: React (TypeScript)
- **Backend**: Python, FastAPI
- **Image analysis**: OpenCV, PIL/Pillow, scikit-image, ExifRead (or piexif)
- **Heatmap generation**: Matplotlib / custom overlay on detected regions

## Core Capabilities (MVP)

1. **Image upload** — accept common formats (JPEG, PNG, WebP, TIFF, BMP) via browser
2. **Manipulation detection** — analyse image for signs of editing, cloning, splicing, compression artefacts; return probability score
3. **Heatmap visualisation** — generate and display a heatmap overlay highlighting manipulated or suspicious regions
4. **EXIF extraction** — parse and return all available EXIF metadata as structured JSON
5. **JSON results export** — return a unified JSON report: `{ score, verdict, heatmap_url, exif, regions }`

## Non-Functional Requirements

- **In-memory processing**: No image data is persisted to disk or cloud storage; all analysis happens in RAM
- **Accuracy**: Detection algorithms must minimise false positives and false negatives; prefer well-established forensic techniques (ELA, noise analysis, clone detection)
- **Performance**: Results should be returned within a reasonable time (target < 10 s for images up to 10 MB)
- **Privacy**: No user data, image content, or EXIF metadata is logged or stored after the request completes
- **Stateless API**: Each request is fully self-contained; no session state on the server

## Standards

### Code Quality
- All Python code must follow PEP 8; use type hints throughout
- FastAPI endpoints must have Pydantic request/response models
- React components must be typed with TypeScript; no `any`
- Unit tests for all detection algorithms; integration test for each API endpoint

### Security
- Validate and sanitise all uploaded files (magic-byte check, size limit)
- Never execute uploaded content; treat all uploads as untrusted data
- No secrets in source code; use environment variables

### API Design
- REST endpoints under `/api/v1/`
- Errors return RFC 7807 Problem Details JSON
- Health check endpoint at `/health`

### Review Criteria
- Manipulation detection must be explainable (heatmap + region annotations)
- EXIF extraction must handle missing/corrupt metadata gracefully
- Frontend must display heatmap inline (no page reload)
